# 意图分析技能 (Intent Analyzer)

## 功能
识别用户输入的主要意图类型，为后续路由提供决策依据。

## 输入
- `userQuery`: string - 用户原始输入

## 输出
```json
{
  "intent": "DATA_QUERY | CHAT | ARTICLE | POLICY | UNKNOWN",
  "confidence": 0.95,
  "reasoning": "用户询问招聘薪资，包含指标关键词"
}
```

## 决策规则

### 1. CHAT (闲聊)
**关键词**: 你好、在吗、谢谢、再见、早上好、help、帮助、能做什么
**示例**:
- "你好"
- "能做什么"
- "谢谢"

### 2. DATA_QUERY (数据查询)
**指标关键词**: 招聘、薪资、工资、企业、公司、数量、岗位、职位、专利、排名、趋势、同比、占比
**地区关键词**: 北京、上海、广州、全国、各省、城市
**时间关键词**: 近、最近、个月、年、趋势

**示例**:
- "北京招聘薪资"
- "各省份企业数量排名"
- "近3个月岗位趋势"

### 3. ARTICLE_QUERY (文章查询) [预留]
**关键词**: 新闻、文章、动态、资讯、报道

### 4. POLICY_QUERY (政策查询) [预留]
**关键词**: 政策、法规、文件、通知、规划

### 5. UNKNOWN (未知)
无法明确分类的输入，需要澄清。

## 处理逻辑
1. 检查是否匹配 CHAT 关键词 → 是则返回 CHAT
2. 检查是否包含指标/数据关键词 → 是则返回 DATA_QUERY
3. 检查是否预留功能关键词 → 返回对应预留类型
4. 以上都不匹配 → 返回 UNKNOWN

## 下游路由
- CHAT → 直接生成友好回复
- DATA_QUERY → 调用 indicator-matcher
- ARTICLE/POLICY → 提示功能开发中
- UNKNOWN → 询问用户具体需求
